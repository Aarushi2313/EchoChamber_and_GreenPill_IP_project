01 / Motivation & Overview

Why We Built This

The Core Question

Social media recommendation algorithms silently shape public opinion at scale. We wanted to experimentally study how platforms like Instagram, YouTube, and Reddit classify and recommend content — and whether mismatched metadata (hashtags vs. actual content) could manipulate algorithmic reach and formation of echo chambers.

Hypothesis

If we manipulate content signals (hashtags, captions, emotional tone) while keeping the underlying content constant, the algorithm will behave differently — creating or breaking echo chambers based on surface-level metadata rather than true content semantics.

Cross-Reel Findings

  • Hashtags alone don't drive reach — trending but irrelevant hashtags performed poorly across all reels
  • Emotional intensity > metadata relevance — highest reach was the violent dog attack clip (199 views) despite mismatched hashtags
  • Storytelling wins on retention — Reel 10 had the lowest skip rate (24%) using a narrative format
  • Polarized content increases reach over neutral content — abortion and voting rights reels outperformed neutral social reels
  • Sensitive social issues get suppressed — feminism and gender reels consistently got very low views
  • Trending audio is supportive but not decisive — several trending-audio reels still performed poorly

Low echo (0.3–0.4): Satirical memes on Trump, Denmark laughs clip

Manual Verification

To validate LLM outputs, we manually classified all 23 reels using the same criteria and compared LLM predictions against actual Instagram metrics. This cross-validation helped identify where the model was accurate and where it diverged from real-world performance — serving as a ground truth check on the AI classifier.

04 / Recommendation Model

BERT Recommendation Model

The Approach

We studied recommendation models in academic literature and identified a BERT-based semantic similarity approach used by real platforms. The model creates dense vector embeddings of all videos and retrieves the most semantically similar ones for any given input.

How It Works

  • Input: A video idea or title as natural language text
  • Embedding: BERT converts all videos into high-dimensional vectors in semantic space
  • Retrieval: Cosine similarity returns the 10 most semantically similar videos
  • Audience Segmentation: Clusters are extracted showing which topical niches the video falls into

Terminal Output Example

Input: "women should get the right to vote" Similar videos retrieved: - Voting Rights for Women - Should Women Lose their Right to Vote? - 'Anti-feminist' says women shouldn't have right to vote - The Female Prophet Vs Andrew Wilson: Should Women Have Voting Rights? [DEBATE] - Defense Secretary Pete Hegseth believes women shouldn't have right to vote? ... Audience segments identified: Segment 1: pete, hegseth, women, defense, secretary Segment 2: women, vote, right, voting, amendment Segment 3: women, vote, right, suffrage, didn

Key Insight

A single video can be simultaneously recommended to multiple ideologically distinct audience segments. This means echo chambers are an emergent property of the embedding space — not explicitly programmed, but arising from semantic proximity.

05 / Literature Review

What the Research Says

Two Rounds of Review

We conducted literature review twice — once before the experiment to design it, and again after to interpret results and build our extension's metrics.

Papers Studied

1. Auditing YouTube's Recommendation Algorithm for Misinformation Filter Bubbles
Misinformation-heavy watch history increases density of false content. The "Up Next" sidebar is the primary driver of echo chambers. Correction is slow — debunking content doesn't immediately neutralize biased history.
2. Degenerate Feedback Loops in Recommender Systems
Mathematically proves that standard collaborative filtering causes user interests to "shrink" over time. Without intentional exploration, systems are destined to narrow a user's world into a degenerate feedback loop.
3. Causally Estimating YouTube's Recommender System Using Counterfactual Bots
Surprising finding: YouTube's algorithm (post-2019) actually acts as a moderator, pulling users toward mainstream content. Echo chambers on YouTube are primarily driven by user choice, not algorithmic entrapment.
4. Echo Chambers on Social Media: A Comparative Analysis
Measures echo chambers via homophily in user interaction networks + bias in information diffusion. Echo chamber strength ∝ similarity(opinions) × interaction frequency.
5. The Echo Chamber Effect on Social Media
Facebook & Twitter → strong echo chambers. Reddit → weaker. Computed user ideology scores, neighbor leaning, community clustering, and diffusion patterns.
6. Echo Chambers in Online Social Networks: A Systematic Literature Review
Uses Echo Chamber Score (ECS) as a silhouette-like metric in BERT embedding space. ECS ≈ 1 = strong echo chamber; ECS < 0 = mixed opinions.

Metrics We Studied & Planned to Implement

  • Modularity (Q) — how well the network splits into separate groups
  • Homophily (H) — edges between similar-opinion nodes / total edges → 1 = echo chamber
  • E-I Index — (External − Internal) / (E + I) → −1 = echo chamber, +1 = neutral
  • Content Diversity Entropy — H = −Σ pᵢ log pᵢ → lower entropy = echo chamber
  • Exposure Diversity — external posts / total posts → low = trapped in echo chamber
  • Assortativity (r) — do similar users connect more than expected by chance?
  • Information Flow / Cascade Analysis — spread_score = communities reached / total communities
  • Bimodal Polarization — Sarle's Bimodality Coefficient, collapse of the "middle ground"
  • Echo Chamber Score (ECS) — silhouette score in BERT embedding space

Composite Neutrality Formula

neutrality_score = w1 × normalized_entropy + w2 × exposure_diversity + w3 × (1 − homophily) + w4 × (1 + EI_index) / 2

Derived from Bakshy et al., Flaxman et al., Garimella et al., and McPherson et al.

06 / Chrome Extension

Breaking the Echo Chamber

What It Does

We built a Chrome Extension that runs on YouTube's homepage to detect echo chambers in real-time and actively recommend diverse content to break the user's filter bubble.

How It Works — Pipeline

  • Step 1 — Scrape: Extension reads the user's current YouTube homepage feed (video titles, thumbnails, channels)
  • Step 2 — Classify: Feed is sent to an LLM which assigns each video an opinion/lean score [0–1] and ideological cluster
  • Step 3 — Threshold: Confidence filter applied — score >0.7 = opinionated, <0.3 = neutral, else discarded
  • Step 4 — Consistency Check: LLM run twice; results discarded if inconsistent (removes noise)
  • Step 5 — Metrics: Entropy, exposure diversity, and homophily computed on the filtered feed
  • Step 6 — Cross-validate: If LLM says "echo chamber" but entropy is high → LLM flagged as wrong
  • Step 7 — Display: Neutrality dashboard shown to user; recommendations provided

Core Features

🔁
Echo Chamber Detector
Real-time banner shown when one-sided content is detected — e.g., "Detected one-sided content: Indian politics and social issues" with Exposure 24%, Diversity 0%, Lean: Left.
🎯
Recommend to Break Echo Chamber
Shows 3–5 videos from opposite or diverse ideological clusters, selected to maximize the entropy of the user's content exposure.
😴
Lazy Mode
User enables lazy mode → recommended diverse videos automatically open in background tabs simultaneously → YouTube feed is reset/refreshed. Zero friction for the user.
📊
Echo Metrics Panel
Visual graphs showing the user's current feed diversity score, exposure score, and a real-time polarization gauge with explanations.

Neutrality Score Formula

neutrality = w1 × normalized_entropy + w2 × exposure_diversity + w3 × (1 − homophily) + w4 × (1 + EI_index) / 2

Robust Pipeline Design

The extension doesn't rely solely on the LLM. It cross-validates LLM outputs against independently computed mathematical metrics. If they disagree, the LLM is flagged as unreliable for that feed. This hybrid approach is grounded in the academic literature on echo chamber measurement.

Baseline Heuristic

If 10+ videos share similar keyword embeddings, the system flags it as suspicious even without LLM confirmation. If the LLM agrees → high confidence echo chamber detected.
